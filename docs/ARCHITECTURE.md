## 核心组件

### 1. Intent Parser (意图识别器)

**职责**: 理解用户输入，识别意图类型和提取实体

**流程**:
1. 关键词快速匹配（高频场景）
2. LLM 深度分析（复杂或模糊场景）
3. 实体提取（项目信息、金额、轮次等）

**输出**:
```typescript
{
  type: 'generate_bp' | 'match_investors' | ...,
  entities: { ... },
  confidence: 0.8
}
```

### 2. Agent 系统

#### BP Agent (商业计划书代理)
- **输入**: 项目基本信息
- **输出**: 结构化的 BP 内容
- **流程**:
  1. 多轮对话收集项目信息
  2. 验证信息完整性
  3. 调用 LLM 生成 BP 内容
  4. 格式化输出
  5. 提供优化建议

#### Financial Agent (财务代理)
- **功能 1**: 财务建模
  - 生成三表（损益表、资产负债表、现金流量表）
  - 计算关键财务指标
  - 敏感性分析

- **功能 2**: 估值分析
  - 多种估值方法（可比、DCF、VC 法、市场倍数）
  - 自动选择合适方法
  - 提供估值区间和依据

#### Investor Match Agent (投资人匹配代理)
- **核心算法**: 多维度匹配打分
  - 融资阶段匹配 (30%)
  - 行业偏好匹配 (30%)
  - 投资金额匹配 (20%)
  - 投资组合相关性 (20%)

- **增强**: LLM 语义理解
  - 投资理念相似度
  - 隐含偏好分析

#### Strategy Agent (策略代理)
- **融资策略**: 轮次规划、时间线、里程碑
- **Term Sheet**: 条款解读、谈判建议
- **尽调准备**: 材料清单、准备指南
- **行业分析**: 趋势研究、竞争格局

### 3. Tools & Utilities (工具层)

#### BP Generator
- Prompt Engineering: 精心设计的系统提示词
- 模板系统: 可扩展的 BP 模板
- 多轮优化: 基于反馈迭代改进

#### Financial Model Builder
- 假设生成: 智能推断业务假设
- 三表联动: 确保财务数据一致性
- 情景分析: 乐观/中性/保守场景

#### Valuation Calculator
- 多方法支持: Comparable, DCF, VC Method, Multiple
- 可比公司查找: LLM + 数据库
- 敏感性分析: 关键参数影响

#### Investor Matcher
- 相似度算法: 多维度加权打分
- 语义匹配: LLM 增强理解
- 推荐解释: 生成匹配理由

### 4. Data Layer (数据层)

#### 投资人数据库
- **结构**: JSON 格式，易于扩展
- **字段**: 基本信息、投资偏好、历史案例、联系方式
- **维护**: 支持增删改查操作

#### 项目信息存储
- **会话状态**: 多轮对话的上下文
- **项目档案**: 完整的项目信息
- **历史记录**: BP 版本、匹配结果等

#### 模板和配置
- **BP 模板**: 可定制的章节模板
- **Prompt 库**: 优化的提示词集合
- **配置参数**: 估值倍数、行业参数等

## 数据流

### 典型场景：生成 BP

```
1. 用户输入
   "帮我生成一份 SaaS 产品的 BP"
         │
         ▼
2. Intent Parser
   - 识别: generate_bp
   - 提取: industry = "SaaS"
         │
         ▼
3. BP Agent
   - 检查现有信息（从会话历史）
   - 如信息不足，发起多轮对话
         │
         ▼
4. 信息收集（多轮）
   "项目名称是什么？"
   "目前有多少用户？"
   "收入情况如何？"
         │
         ▼
5. 信息验证
   - 检查必填字段
   - 验证数据合理性
         │
         ▼
6. BP 生成
   - 构建 Prompt
   - 调用 LLM
   - 解析结构化内容
         │
         ▼
7. 格式化输出
   - 章节摘要
   - 关键数据
   - 优化建议
         │
         ▼
8. 返回用户
   通过原渠道发送结果
```

### 典型场景：匹配投资人

```
1. 用户输入
   "推荐适合 Pre-A 轮的投资机构"
         │
         ▼
2. Intent Parser
   - 识别: match_investors
   - 提取: fundingRound = "pre-a"
         │
         ▼
3. Investor Match Agent
   - 获取项目信息
   - 加载投资人数据库
         │
         ▼
4. 匹配计算
   - 阶段匹配打分
   - 行业匹配打分
   - 金额匹配打分
   - 组合相关性打分
         │
         ▼
5. 排序和筛选
   - 按总分排序
   - 过滤低分（<30%）
   - 取 Top 10
         │
         ▼
6. 生成推荐理由
   - 匹配点说明
   - 接触建议
         │
         ▼
7. 格式化输出
   - 机构卡片
   - 匹配度展示
   - 联系方式
         │
         ▼
8. 返回用户
```

## 性能优化

### 1. 缓存策略
- LLM 响应缓存（相似问题）
- 投资人数据缓存
- 行业数据缓存

### 2. 并行处理
- 多个独立查询并行执行
- 批量数据加载

### 3. 渐进式响应
- 先返回简要结果
- 后台继续生成详细内容

## 安全性

### 1. 数据隐私
- 敏感信息加密存储
- 会话隔离

### 2. 输入验证
- 防止注入攻击
- 数据格式验证

### 3. 访问控制

- API 调用限流
